架設Hexo的部落格及自動發布到Github

 發表於 October 16, 2020  |  2 分鐘  |  277 個字  |  Aaron  • 其他語言: English

🍅簡易安裝架設Hexo的部落格及自動發布到Github，順便記錄架設部落格的過程。

git: 2.18.4-2.el8_2
git: 2.24.1.windows.2
hexo: 5.2.0
hexo-cli: 4.2.0
CentOS: 8.2.2004
Windows: 10 1909

🍐安裝Hexo

1. 檢查git、node及npm是否有安裝

git --version
node -v
npm -v

2. 升級npm版本 (選擇)

npm install npm -g

3. 安裝 hexo-cli hexo-admin

npm install hexo-cli hexo-admin -g

4. 檢查hexo版本

hexo version

🍑設定Hexo

1. 建立部落格網站

hexo init <sitename>

2. 新增主題 (git clone)

git clone https://github.com/next-theme/hexo-theme-next themes/next

3. 應用主題及修改配置文件

  • 配置文件路徑及名稱: <sitename>/_config.yml

  • 修改主題設定

theme: next

4. 安裝hexo的插入圖片工具

npm install hexo-asset-image --save

5. 應用插入圖片功能

  • 配置文件路徑及名稱: <sitename>/_config.yml
  • 修改文章的asset設定
post_asset_folder: true

6. 自動安裝兼容更新及脆弱依賴

npm audit fix

7. 啓動Hugo並生成靜態網站文件

hexo clean
hexo s -g

8. 檢查網頁

  • Hexo頁面: http://localhost:4000/

  • Hexo-admin頁面: http://localhost:4000/admin/

🍆自動發布到Github

1. 安裝hexo的自動發布到Git的工具

npm install hexo-deployer-git --save

2. 設定自動發布插件

  • 配置文件路徑及名稱: <sitename>/_config.yml
  • 新增以下內容
deploy:
  type: git
  repository: git@github.com:<username>/<repository_name>.git
  branch: gh-pages

🍇文章設定

1. 設定文章儲存的路徑

  • 配置文件路徑及名稱: <sitename>/_config.yml
  • 修改new_post_name的設定值
new_post_name: :year/:title.md

2. 設定文章橂板

  • 配置文件路徑及名稱: <sitename>/scaffolds/post.md
  • 修改permalink的設定值
permalink: {{ title }}

🍉Flowchart插件

1. 安裝flowchart插件

npm install --save hexo-filter-flowchart

2. 設定flowchart插件

  • 配置文件路徑及名稱: <sitename>/_config.yml
  • 新增以下內容
flowchart:
  # raphael:   # optional, the source url of raphael.js
  # flowchart: # optional, the source url of flowchart.js
  options: # options used for `drawSVG`

🍌Mermaid插件

1. 安裝mermaid插件

npm install hexo-filter-mermaid-diagrams

2.1. 設定mermaid插件

  • 配置文件路徑及名稱: <sitename>/_config.yml
  • 新增以下內容
# mermaid chart
mermaid: ## mermaid url https://github.com/knsv/mermaid
  enable: true  # default true
  version: "7.1.2" # default v7.1.2
  options:  # find more api options from https://github.com/knsv/mermaid/blob/master/src/mermaidAPI.js
    #startOnload: true  // default true
  • 配置文件路徑及名稱: <sitename>/themes/next/layout/_partials/footer.swig
  • 新增以下內容
{% if theme.mermaid.enable %}
  <script src='https://unpkg.com/mermaid@{{ theme.mermaid.version }}/dist/mermaid.min.js'></script>
  <script>
    if (window.mermaid) {
      mermaid.initialize({{ JSON.stringify(theme.mermaid.options) }});
    }
  </script>
{% endif %}

2.2. 設定mermaid插件 (for next theme)

  • 配置文件路徑及名稱: <sitename>/themes/next/_config.yml
  • 在檔案中尋找 # mermaid chart
  • 修改以下設定值
# mermaid chart
mermaid: ## mermaid url https://github.com/knsv/mermaid
  enable: true  # default true
  version: "7.1.2" # default v7.1.2
  options:  # find more api options from https://github.com/knsv/mermaid/blob/master/src/mermaidAPI.js
    #startOnload: true  // default true

完
